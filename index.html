"patrón modulo"

referencia a otros módulos se hace por variables globales

se mancha el espacio de nombres global

el orden de referencias lo tenés que resolver vos

(ej una biblioteca que usa jQuery tiene que estar despues del tag que carga jquery)

optimizar para producción se complica

necesitás herramientas para pasar de varios tags a un solo tag

require.js

implementa la especificación AMD

cargador de modulos (y archivos)

resuelve dependencias y las carga de manera asíncrona

está optimizado para el lado cliente, navegador

cómo se define un módulo


define(id?, dependencias?, fabrica);

especificación AMD


define(['app/ui/button'], function (Button) {

   var ui = {};

   ui.buttons = [new Button()];

   return ui;

});

cada función definida por define se llama cuando sus dependencias son cargadas
